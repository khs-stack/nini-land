# PROJECT_SPEC.md — 기능정의서

## 1. 프로젝트 개요

NiNi Land가 직접 디자인·생산한 아동복을 일반 소비자와 도매 사업자에게 판매하는
NiNi Land 전용 도·소매 통합 쇼핑몰. 모바일 우선 반응형 웹앱으로 시작하며, 자체 주문/결제
기능을 초기 버전부터 포함한다(외부몰 단순 이동 구조 아님). 추후 앱 전환, 네이버 플레이스/
네이버페이 등 네이버 관련 서비스 연동을 고려해 확장 가능하게 설계한다.

### 회원 유형 정의 요약

- **비회원**: 일반 소비자로 취급, 도매가 비노출, 비회원 구매 가능
- **일반회원**: 카카오/네이버/휴대폰·이메일 가입, 소비자가 기준 전체 구매 기능
- **사업자 승인대기 회원**: 사업자 정보 제출 완료, 관리자 승인 전까지 소비자가로만 구매 가능
- **승인된 도매회원**: 관리자 승인 후 도매가/도매 주문/이미지 일괄저장 등 전체 도매 기능 이용
- **관리자**: 전체 기능 및 정보 열람·수정 권한

## 3. 회원 유형별 가격 노출 규칙

원칙: **가격 숨김은 화면 렌더링(CSS/조건부 렌더)이 아니라 서버 응답 자체에서 결정한다.**
승인된 도매회원 여부를 서버에서 확인한 뒤에만 도매가 데이터를 API 응답에 포함시키며,
비인가 사용자의 API 응답·페이지 소스·개발자도구에는 도매가 필드 자체가 존재하지 않아야 한다.

| 회원 유형 | 노출 정보 | 비고 |
|---|---|---|
| 비회원 / 일반회원 | 소비자가, 할인 소비자가, `사업자 전용 도매가 별도` 배지, 안내 문구, 도매회원 가입 버튼 | 도매가를 `비공개/문의/0원` 등으로 표기하지 않음 |
| 사업자 승인대기 | 소비자가, "사업자 인증 확인 중" 안내, 승인 후 도매가 확인 가능 안내 | 소비자가로 구매만 가능 |
| 승인된 도매회원 | 도매가, 권장 소비자가, 최소주문수량, 주문단위, 컬러/사이즈별 주문조건, 혼합주문 가능여부, 부가세 포함/별도 | 서버에서 승인 상태 검증 후 응답 |

안내 문구(비회원/일반회원 공통 표시):

> 현재 표시된 가격은 일반 소비자 판매가격입니다.
> 사업자 회원은 사업자 인증 및 관리자 승인 후 별도의 도매가격을 확인할 수 있습니다.

## 4. 공구기간 및 배송일정 상태값

공구 예정 → 공구 진행 중 → 주문 마감 → 제작 중 → 입고 예정 → 배송 준비 중 → 배송 중 → 공구 종료

상품별 등록 항목: 공구 시작일/종료일, 주문 마감일, 예상 배송 시작/완료일, 리오더 예정일,
입고 예정일, 배송 지연 안내, 공구 상태, 추가 안내문. 목록/상세 페이지에 배지로 표시.

## 5. 주문 상태값

결제대기 · 입금대기 · 결제완료 · 상품준비중 · 제작중 · 배송준비중 · 배송중 · 배송완료 ·
취소요청 · 취소완료 · 교환요청 · 교환진행중 · 반품요청 · 반품진행중 · 환불완료

일반 주문과 도매 주문은 구분 관리하며, 관리자 목록은 전체/일반/도매/공구/결제상태/배송상태/
주문기간/상품명/회원명/상호명/사업자등록번호로 필터링 가능해야 한다.

## 6. 문의 연결 요구사항

- 카카오톡 문의(아이디는 관리자 설정값, 코드 하드코딩 금지), 카카오 오픈채팅, 상품 링크 복사,
  도매 문의하기, 일반 상품 문의하기
- 도매회원 문의 시 상품명·상품번호·선택 옵션·문의유형·현재 페이지 주소를 함께 전달

## 7. 네이버/외부 서비스 연동 (확장 대비)

초기 개발부터 관리자 설정값으로 준비: 네이버 플레이스 주소, 스마트스토어 주소, 네이버 톡톡
주소, 네이버 로그인 설정, 네이버페이/결제 연동 설정값, 외부 상품 주소, 외부 채널 사용 여부.
연동 코드는 별도 모듈로 분리하며, 자체 주문 기능을 대체하거나 제거하지 않는다.
