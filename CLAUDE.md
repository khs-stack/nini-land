# CLAUDE.md — NiNi Land 도·소매 통합 쇼핑몰

이 문서는 이 저장소에서 작업하는 개발자(및 AI 어시스턴트)를 위한 최상위 가이드입니다.
코드를 수정하기 전에 이 문서와 `docs/` 폴더의 문서를 먼저 읽으십시오.

## 프로젝트 성격

- 여러 업체가 입점하는 오픈마켓/도소매 커뮤니티가 **아닙니다.**
- NiNi Land가 직접 디자인·생산한 아동복을 **일반 소비자 + 도매 사업자**에게 판매하는
  **NiNi Land 단독 쇼핑몰**입니다.
- 모바일 우선(mobile-first) 반응형, 초기에는 모바일 웹앱, 추후 앱 전환 및
  네이버 플레이스/네이버 서비스 연동을 고려한 확장 가능한 구조로 설계합니다.

## 기술 스택

| 영역 | 기술 |
|---|---|
| 프레임워크 | Next.js (App Router) |
| 언어 | TypeScript |
| 백엔드/DB | Supabase (Postgres + Auth + Storage + RLS) |
| 배포 | Vercel |
| 스타일 | 모바일 퍼스트 반응형 |
| 관리자 | 별도 관리자 페이지 포함 |

## 작업 원칙

1. **기존 프로젝트 우선 확인.** 새 프로젝트를 함부로 만들지 않는다. 이번 세션은 기존 저장소가
   없는 상태로 확인되어 `docs/` 기획 문서부터 신규 작성했다.
2. **문서 우선.** 코드를 수정하기 전 `docs/PROJECT_SPEC.md`, `docs/SCREEN_FLOW.md`,
   `docs/DB_SCHEMA.md`, `docs/TASKS.md`, `docs/DECISIONS.md`, `docs/SECURITY_CHECKLIST.md`,
   `docs/BUG_LOG.md`를 먼저 읽는다.
3. **기능 축소 금지.** 기존에 정의된 기능이나 화면을 임의로 삭제·간소화하지 않는다. 범위를
   줄여야 한다면 `docs/DECISIONS.md`에 사유를 기록하고 사용자 승인을 받는다.
4. **단계적 개발.** `docs/TASKS.md`의 개발 단계를 순서대로 진행하며, 각 단계 완료 시
   테스트 후 Git 커밋을 남긴다.
5. **가격/권한 로직은 반드시 서버에서 검증.** 도매가 노출 여부, 관리자 권한, 결제 금액은
   클라이언트 값을 신뢰하지 않고 서버(Supabase RLS + API 레이어)에서 재검증한다.
6. **모듈 분리.** 결제(PG) 모듈, 네이버/카카오 등 외부 서비스 연동 모듈은 특정 벤더에 종속되지
   않도록 별도 모듈로 분리한다.

## 코딩 전 보고 절차

새로운 기능을 구현하기 전에는 다음을 먼저 보고하고 승인을 받는다.

1. 현재 프로젝트 구조
2. 기존 구현 상태
3. 새로 추가/수정할 기능
4. 수정할 파일 목록
5. 데이터베이스 변경 사항
6. 개발 순서
7. 예상되는 문제
8. 보안상 주의할 부분
