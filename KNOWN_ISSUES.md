# KNOWN_ISSUES — 확인된 문제와 우선순위

## P0/P1: 먼저 해결

### 1. 상품 옵션 선택이 되지 않거나 선택 상태가 불명확
- 사용자 보고: “선택이 안되는데”
- 컬러, 사이즈, 수량 선택의 state·disabled·재고·포커스·시각 상태 전수 확인
- 선택 전 장바구니/바로주문 차단
- 잘못된 조합 및 품절 처리
- 모바일 터치 영역 확인

### 2. 상품 이미지가 깨지거나 심하게 확대/픽셀화
- 화면: `screenshots/05_product_image_bad_quality_and_layout.png`
- 단순 해상도 수치만 확인하지 말고 실제 이미지 콘텐츠와 CSS 렌더링을 눈으로 검증
- 대표사진과 상품명이 맞는지 확인
- 원본 사진 ZIP과 public WebP 변환본 비교
- `object-fit`, aspect-ratio, max-height, responsive sizes, Next/Image 설정 검토
- 상세 대표이미지가 화면 전체를 비정상적으로 차지하지 않게 수정
- 샘플/placeholder로 숨기지 말고 실제 판매 사진을 정상 연결

### 3. 전체보기/신상품/베스트/공구에 상품이 보이지 않음
- 사용자 보고: 전체보기, 신상품, 베스트 진입 시 상품 미표시
- 카테고리 slug/한글 이름/필터 조건/URL decoding/seed 데이터 불일치 확인
- 홈 카드 링크와 분류 페이지 경로 일치
- 빈 상태가 잘못 뜨지 않는지 확인
- 5개 상품이 의도한 분류에 나타나는지 명시적 테스트

### 4. 주문 방법이 불명확하고 구매 흐름이 끊김
- 화면: `screenshots/06_order_option_flow_missing.png`
- 상품 상세에서 컬러/사이즈/수량 선택 → 장바구니/바로주문 → 주문서 → 완료 → 상세까지 연결
- 승인 도매회원은 최소수량과 주문단위 검사
- 일반회원은 소비자가 주문
- 버튼 문구와 disabled 이유를 사용자에게 표시
- 실패 시 toast 또는 인라인 에러로 원인 표시

### 5. 예전 상품 링크가 “상품을 찾을 수 없습니다”
- 화면: `screenshots/04_old_product_not_found.png`
- 기존 slug를 삭제했으면 redirect/alias 또는 친절한 대체 상품 안내
- 북마크·최근본상품·장바구니의 오래된 productId 안전 처리
- 기존 기능 삭제 금지 원칙에 따라 단순 404로 방치하지 말 것

## P1: 데이터·권한·가격

### 6. 역할 변경 시 장바구니 가격 오염 가능
- 도매회원으로 담은 도매가 cart item이 일반회원/비회원에게 남지 않도록 검사
- 로그인/로그아웃/setDemoRole 시 장바구니 처리 원칙 통일
- 주문 시 현재 role과 현재 상품 데이터로 재계산

### 7. 무료배송 기준 계산 불일치 가능
- 장바구니 표시, 주문서, 주문 생성 로직에서 동일한 기준 사용
- 일반/도매 배송비 정책을 분리
- 입력 subtotal을 신뢰하지 말고 createOrder 내부에서 재계산

### 8. 도매 최소수량과 옵션 재고 충돌
- 최소수량보다 재고가 적을 때 주문 불가 이유 표시
- 혼합 주문 가능 상품은 옵션 합산 기준 명확화
- orderUnit 배수 검사
- 수량 버튼의 min/max와 실제 검증 일치

### 9. 추측 도매가 확정 표시
- 레고/룩앤미/김장룩 도매가는 ChatGPT가 임시로 가정한 값
- 관리자 목업에서는 `임시` 또는 확인 필요 상태로 관리하거나 사용자 확정 전 실판매 숫자로 간주하지 말 것
- 로카나시세트 도매가 14,000원만 사용자 제공으로 확인됨

### 10. 일반회원 도매가 비노출 검증
- DOM, 화면, 상태 전달, 직렬화 데이터, 개발자도구 관점에서 점검
- 현재 localStorage에 전체 상품 객체가 저장되면 도매가가 포함될 수 있음. 목업 단계의 한계와 개선 가능한 분리 구조 검토
- UI CSS 숨김만으로 해결하지 말 것

## P1/P2: React 상태와 데이터

### 11. 최근 본 상품 effect 반복 가능성
- 상세 페이지에서 `addRecentlyViewed`가 렌더마다 반복되지 않도록 의존성 안정화
- 오래된 id 제거

### 12. 목업 store seed 버전 마이그레이션
- 모든 상태를 무조건 초기화하지 않기
- products/cart/wishlist/recent/orders/addresses를 각각 정책 분리
- 스키마가 바뀐 cart item 안전 변환

### 13. createOrder 실패 이유 미표시
- null만 반환하지 말고 결과 타입 또는 검증 오류 제공
- 재고, 옵션, 역할, 수량, 주소, 결제수단별 메시지

### 14. 주문 시점 가격 스냅샷과 상품 가격 재계산
- 주문 직전 현재 상품/역할로 서버 역할 로직에서 재계산
- order item에 unit price, price type, 옵션, 상품명, 이미지 스냅샷
- 이후 상품 수정이 과거 주문에 영향 없음

## P2: 관리자

### 15. 관리자 상품 관리
- 실제 5개 상품 옵션·이미지·분류·badge·가격·재고 편집 가능
- 이미지 순서/대표이미지 지정
- 새 상품/베스트/공구 여부를 category 문자열 하나에 과도하게 의존하지 않도록 개선 검토

### 16. 관리자 주문 관리
- 주문 상세, 상태 변경, 운송장/택배사, 메모, 일반/도매 구분
- 모바일 표 깨짐 방지
- 상태 변경 후 고객 주문 상세 즉시 반영

### 17. 회원 승인
- pending → approved/rejected 흐름
- 관리자만 변경
- 승인 후 가격 화면 변화
- 사업자 정보가 없는 상태 예외처리

### 18. 공지/팝업/문의/반품
- 버튼만 존재하고 저장/수정/삭제/목록 반영이 안 되는 항목 탐색
- localStorage persistence 확인
