1. 저장소의 CLAUDE.md
2. PROJECT_SPEC.md, SCREEN_FLOW.md, DB_SCHEMA.md

현재 프로젝트는 Next.js App Router + TypeScript이며 실제 루트는 app/이다.
Supabase 연결 전 목업 단계이고 app/src/app/lib/mockStore.tsx의 React Context + localStorage를 사용한다.

중요한 현재 상태:
- GitHub/Vercel에는 실제 상품 업데이트와 HeroSlide 빌드 오류 수정까지만 반영된 것으로 보인다.
- `niniland_full_fix_NOT_APPLIED.zip`은 ChatGPT 후보 수정본이며 사용자가 아직 적용하지 않았다.
- 후보 ZIP을 통째로 덮어쓰지 말고 현재 git 코드와 비교해 필요한 변경만 선별 반영해라.
- 새 프로젝트, 별도 사본 배포, 별도 Vercel 링크를 만들지 마라.
- 기존 기능을 삭제하거나 축소하지 마라.
- 사용자가 명시적으로 요청하기 전 Supabase, PG, 실제 카카오/네이버 로그인은 연결하지 마라.
- 일반/비회원/승인대기에게 도매가 숫자나 필드를 노출하지 마라.
- 사용자는 비개발자이므로 모든 파일 위치는 app/src/app/... 전체 경로로 말하고 명령은 한 줄씩 줘라.

사용자가 보고한 실제 문제:
- 컬러/사이즈 선택이 안 됨
- 사진이 깨지거나 너무 확대되고 상품과 사진이 안 맞아 보임
- 전체보기/신상품/베스트/공구 화면에 상품이 안 보임
- 주문 방법이 불명확하고 옵션 선택 뒤 주문 흐름이 없음
- 기존 상품 링크가 상품 없음으로 뜸
- 전체적으로 클릭 테스트 없이 배포된 상태

작업 방식:
1. 먼저 git status, git log -8 --oneline, 현재 파일 구조를 확인해 실제 상태를 보고해라.
2. npm run lint, npx tsc --noEmit, npm run build를 실행해 기준선을 잡아라.
3. npm run dev로 실행하고 홈부터 상품/카테고리/검색/상세/장바구니/주문/마이/관리자까지 실제 클릭 테스트해라.
4. QA_320_CHECKLIST.md 320항목을 PASS/FIX/NA로 채우고, 문제는 재현경로·원인·수정파일·검증결과를 기록해라.
5. 치명/높음 문제를 먼저 고친 뒤 디자인·모바일·접근성·관리자 개선을 진행해라.
6. 원본 상품 사진 ZIP을 직접 확인해 상품명과 사진을 다시 매칭하고 실제 페이지에서 품질을 검증해라.
7. 레고/룩앤미/김장룩 도매가는 사용자 확정값이 아니므로 확정 가격처럼 취급하지 마라. 로카 도매가 14,000원만 확정이다.
8. 역할 변경, localStorage seed 버전, 주문 가격 재계산, 최소수량/재고, 무료배송 계산을 집중 검증해라.
9. 큰 작업 전 수정할 파일과 위험을 짧게 보고하되, 사용자가 이미 전체 점검과 수정을 승인했으므로 불필요하게 매 단계 승인을 기다리지 말고 계속 진행해라.
10. 기능별로 의미 있는 커밋을 만들고 최종에 lint/ts/build 및 핵심 사용자 흐름을 다시 테스트해라.
