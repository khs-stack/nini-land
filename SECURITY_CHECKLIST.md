# SECURITY_CHECKLIST.md — 보안 검수 목록

각 항목은 코드 리뷰 및 배포 전 체크리스트로 사용한다.

- [ ] 일반회원/비회원 대상 API 응답에 도매가 필드가 포함되지 않는다 (필드 자체를 제외, 마스킹 아님)
- [ ] 도매가 접근은 Supabase RLS로 `role = wholesale_approved` 또는 `admin`만 허용
- [ ] 사업자 승인상태(`approval_status`)는 사용자가 직접 변경할 수 없다 (UPDATE 권한 admin 한정)
- [ ] 사업자 승인/거절 액션은 서버에서 admin role 검증 후에만 처리
- [ ] 사업자등록증 등 서류는 공개 Storage 버킷에 두지 않는다 (비공개 버킷 + signed URL)
- [ ] 사업자등록증 열람 권한은 관리자와 해당 신청자 본인으로 제한 (RLS)
- [ ] 결제 금액은 클라이언트 전달값을 신뢰하지 않고 서버에서 상품/옵션 가격 기준으로 재계산
- [ ] 주문 시점 가격(`price_at_order`)을 order_items에 스냅샷으로 별도 저장
- [ ] 재고 차감은 트랜잭션/원자적 연산으로 처리하여 중복 차감(동시 주문 경합) 방지
- [ ] 관리자 페이지 접근은 클라이언트 라우팅 가드뿐 아니라 서버(미들웨어/API)에서도 role 검증
- [ ] 카카오/네이버 API 키, PG 키 등은 모두 환경변수로 관리하고 클라이언트 번들에 노출하지 않는다
- [ ] 도매가/사업자 정보 관련 페이지 소스, 브라우저 개발자도구 Network 탭에서 비인가 사용자가
      도매가를 확인할 수 없는지 실제로 검증 (수동 QA 필수)
- [ ] 게스트/세션 기반 장바구니에 타 사용자 데이터가 섞이지 않도록 세션 격리 확인
- [ ] 결제 웹훅/콜백은 서명 검증 없이 주문 상태를 변경하지 않는다 (PG 모듈 구현 시 확인)
- [ ] 관리자 설정값(카카오ID, 오픈채팅 링크 등)은 코드에 하드코딩하지 않고 DB/설정 테이블에서 로드
