# SCREEN_FLOW.md — 화면 목록 및 구매 흐름

## 1. 일반 소비자 구매 흐름

```
상품 선택
  → 컬러·사이즈 선택
  → 장바구니 담기 또는 바로구매
  → 배송지 입력 (배송비 자동 계산)
  → 쿠폰/포인트 적용(선택)
  → 결제 (카드결제 / 간편결제 확장 구조, 초기엔 테스트 결제 환경)
  → 주문완료
  → 주문내역 / 배송조회
  → (필요시) 취소·교환·반품 요청 → 환불 처리
```

부가 요소: 현금영수증 처리 구조, 결제 영수증 확인, 품절 처리, 재고 차감(중복 차감 방지).

## 2. 도매회원 구매 흐름

```
로그인 (승인된 도매회원)
  → 도매가 확인 (서버 인증 후 노출)
  → 옵션/컬러/사이즈별 수량 입력
  → 최소 주문수량 / 주문단위 / 혼합주문 가능여부 검사
  → 도매 장바구니
  → 주문서 (사업자정보 자동 적용)
  → 도매 배송비 계산
  → 결제 (카드결제 또는 무통장입금 선택)
  → 주문완료 (거래명세 관련 정보 저장)
  → 주문내역 / 배송조회
  → (필요시) 취소·반품 요청 → 관리자 확인·처리
```

카카오톡 문의 버튼은 주문 기능과 별도로 항상 유지(상담용).
