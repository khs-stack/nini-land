## 이번 프로젝트에서 첫 번째로 구현할 기능 제안

**제안: 2단계 중 "회원 유형별 가격 표시 분기가 가능한 상품 상세 화면"부터 시작**

이유:
1. 이 프로젝트의 핵심 차별점(소비자가/도매가 이원화, 서버 기반 접근 제어)이 가장 먼저
   검증되어야 이후 장바구니·주문·결제 로직이 안전하게 그 위에 쌓일 수 있습니다.
2. Next.js + Supabase 기본 연결, Auth 없이도(우선 role을 임시 파라미터로 시뮬레이션)
   UI 레이아웃과 가격 노출 규칙을 먼저 확정할 수 있어 이후 단계 작업 속도가 빨라집니다.
3. `product_consumer_price` / `product_wholesale_price` 테이블 구조와 RLS 정책을 이 단계에서
   미리 설계해두면 3~4단계(인증, 관리자 상품등록)의 스키마 변경을 줄일 수 있습니다.

구체적 착수 순서 제안:
1. Next.js 프로젝트 생성 + Supabase 연결
2. `products`, `product_options`, `product_consumer_price`, `product_wholesale_price` 테이블 생성
3. 상품 목록/상세 화면 (역할별 더미 세션으로 가격 분기 렌더링 확인)
4. 이후 3단계(인증)로 진행하여 실제 role 기반 접근 제어로 교체
